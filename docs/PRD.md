# 🌙 [기획서] DreamLog (가제) - 웹 MVP 통합 PRD & UI/UX 설계서

> **문서 버전**: v1.0 (최종 통합본)
> 
> 
> **플랫폼**: 모바일 웹 (Mobile Web MVP) - *안티그래비티 기반 구축*
> 
> **디자인 테마**: 다크 모드(밤하늘/네온) 기본, 라이트 모드(파스텔) 토글 지원
> 
> **핵심 페르소나**: 고양이 타로 마스터 **'몽냥이'** ("~다 냥!", 친근한 구어체)
> 

---

## 💡 1. 서비스 개요 및 목표

- **기획 배경**: 기상 직후 휘발되는 꿈을 기록하고, 그 의미를 알고 싶어 하는 심리적/예지적 니즈 충족.
- **서비스 한 줄 요약**: "잊기 전 기록하고, 몽냥이와 집단지성으로 해석하는 꿈 관리 플랫폼"
- **핵심 가치**:
    1. 빠른 기록 (휘발 방지)
    2. 즉각적이고 감성적인 AI 해몽 (몽냥이 페르소나)
    3. 익명 기반의 꿈 공유 및 소통 (집단지성)

## 💰 2. 비즈니스 및 운영 정책

- **수익 모델 (Monetization)**:
    - [기본 AI 해몽 요약]: 전체 무료 제공 (총평 및 자동 태그).
    - [심층 AI 해몽 리포트]: **보상형 비디오 광고(Video AD) 시청 시 잠금 해제** (재물운, 연애운, 조언 등).
- **회원가입/로그인 (Gated Content)**:
    - OAuth 2.0 소셜 로그인 (카카오톡, 구글). *이메일 통합 없이 진입 채널별 독립 계정 처리.*
    - **비회원 권한**: 메인/커뮤니티 글 목록(제목)만 열람 가능. 상세 진입 시 본문 **블러(Blur) 처리** 및 로그인 강제.
- **커뮤니티 게이미피케이션**:
    - 익명 활동. 댓글 작성 및 '질문자 채택' 수에 따라 유저 등급 상승.
    - 등급: Lv.1(꿈나무) -> Lv.2(해몽 지망생) -> Lv.3(전문 해몽가 뱃지 부여).

## 🚨 5. 기획 누락 방지 & 개발자 전달 노트 (Dev Notes)

안티그래비티 기반 모바일 웹 구축 시, 개발팀에서 반드시 체크해야 할 사항입니다.

1. **상태 이탈 방지 (작성 폼)**:
    - 유저가 꿈을 길게 작성하다가 브라우저 '뒤로 가기'나 스와이프를 할 경우, `beforeunload` 이벤트 등을 활용해 **"작성 중인 내용이 사라집니다. 나가시겠습니까?"** 모달을 띄워 데이터 날림을 방지해야 합니다.
2. **로딩 애니메이션 (AI 분석 대기)**:
    - `[몽냥이에게 해몽 받기]` 버튼 클릭 시 LLM API 통신으로 인한 지연 시간(3~8초)이 발생합니다. 이때 화면이 멈춘 것처럼 보이지 않도록 **"몽냥이가 열심히 타로 카드를 섞고 있다 냥! 🐾"** 라는 Lottie 애니메이션 또는 텍스트 로딩 화면이 필수입니다.
3. **블러(Blur) 해제 트랜지션**:
    - 보상형 광고 시청 완료 후 콜백(Callback)을 받으면, CSS `filter: blur()` 값을 0으로 부드럽게(`transition`) 빼주면서 파티클 라이브러리(예: canvas-confetti)를 호출해 시각적 쾌감을 줍니다.
4. **인앱 알림 센터 (GNB 우상단 🔔)**:
    - 웹 환경이라 OS 푸시 알림은 안 되지만, 내 꿈에 댓글이 달리거나 내 댓글이 '채택'되었을 때 DB에 알림 데이터를 쌓아둡니다. 유저가 웹에 재접속했을 때 종 모양 아이콘에 **빨간색 뱃지(N)**를 띄워 리텐션을 유도합니다.
5. **로그인 리다이렉트 (Deep Link)**:
    - 비회원이 특정 커뮤니티 글을 보려고 로그인을 수행한 후, 메인(Home)으로 튕기지 않고 **원래 보려던 글의 상세 페이지로 정확히 리다이렉트** 시켜주어야 합니다.
